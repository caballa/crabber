# crabber.wasm — serverless CrabIR playground

A WebAssembly build of `crabber`. It runs the full CrabIR parser + Crab analyzer
entirely in the browser or in Node — **no server, no backend**. A playground is
just static files (`crabber.js`, `crabber.wasm`, `index.html`) served from any
static host (GitHub Pages, Netlify, `python3 -m http.server`, …).

## What's here

| File | Purpose |
|------|---------|
| `build.sh`        | Build `crabber.js` + `crabber.wasm` (main entry point). |
| `build-gmp.sh`    | One-time cross-compile of GMP to WASM (auto-run by `build.sh`). |
| `wasm-config/`    | WASM `crab/config.h` (all optional backends disabled). |
| `gen-examples.py` | Regenerate `examples.js` (the CTest programs) from `../samples`. |
| `run.mjs`         | Node CLI runner. |
| `index.html`      | Minimal browser playground (with syntax highlighting + example loader). |
| `examples.js`     | Generated: the CTest sample programs + their domains/flags, for the in-page "Example" menu. |
| `crabber.js/.wasm`| Build outputs (regenerated by `build.sh`). |
| `gmp-wasm/`       | GMP compiled to WASM (produced by `build-gmp.sh`). |
| `lib/`            | `libcrab_wasm.a` (produced by `build.sh`). |

## Prerequisites

1. **Emscripten SDK.** Install once, then activate it in every shell you build from:
   ```bash
   git clone https://github.com/emscripten-core/emsdk.git ~/emsdk
   cd ~/emsdk
   # emsdk needs Python >= 3.10; point it at one if your default is older:
   EMSDK_PYTHON=$(which python3.12) ./emsdk install latest
   ./emsdk activate latest
   ```
   Then in each build shell:
   ```bash
   source ~/emsdk/emsdk_env.sh
   ```

2. **Boost headers.** Header-only is enough (Crab uses no compiled Boost libs).
   Default lookup is Homebrew's `/opt/homebrew/include`; override with
   `BOOST_INC=/path/to/include` if elsewhere.

3. **Crab source tree.** Expected at `../crab` relative to the repo root; override
   with `CRAB_ROOT=/path/to/crab`. No native Crab *build/install* is required —
   this compiles straight from Crab's source.

## Build

```bash
source ~/emsdk/emsdk_env.sh
./build.sh
```

The first run also cross-compiles GMP to WASM (a few minutes); later runs reuse
`gmp-wasm/`. Outputs: `crabber.js` and `crabber.wasm` in this directory.

Path overrides (all optional):
```bash
CRAB_ROOT=/path/to/crab BOOST_INC=/opt/homebrew/include ./build.sh
```

## Run — Node

```bash
node run.mjs ../samples/test-1.crabir -d int --print-invariants
```
Any `crabber` flag works (`-d <domain>`, `--print-invariants`, `--widening-delay`,
`--coefficients`, …); `--show-domains` lists them.

## Run — browser (locally)

Serve this directory (the `.wasm` must be fetched over HTTP, not `file://`):
```bash
python3 -m http.server 8000
# open http://localhost:8000/index.html
```

## Publish on GitHub Pages (no CI)

The playground is static files, so GitHub Pages serves it directly. The repo's
`docs/` folder holds the published copy. Our build is single-threaded (no
`SharedArrayBuffer`), so no special COOP/COEP headers are needed and plain Pages
works; `.wasm` is served with the correct `application/wasm` MIME type.

1. Refresh `docs/` from the current build:
   ```bash
   ./deploy-docs.sh          # copies index.html + crabber.js + crabber.wasm into ../docs
   ```
2. Commit and push:
   ```bash
   git add docs && git commit -m 'Update WASM playground' && git push
   ```
3. One-time: GitHub → repo **Settings → Pages → Source: Deploy from a branch**,
   branch `main`, folder `/docs`. Save.

Live at **https://caballa.github.io/crabber/** (first publish takes a minute).

Note: `docs/crabber.wasm` (~3.9 MB) is versioned in git; rerun `deploy-docs.sh`
and recommit whenever you rebuild.

## Supported abstract domains

Working in WASM (pure C++):
`int`, `dis-int`, `int-terms`, `int-set`, `int-val-part`,
`zones`, `zones-val-part`, `oct-snf`, `non-unit-oct`.

Excluded — need external C libraries not ported here (selecting one prints a
clean "not registered" message):
`boxes` (LDD), `oct` / `pk` (Apron/Elina), `pk-pplite` (PPLite).

## How it works (notes for future maintenance)

- **GMP** is Crab's one hard C dependency (`z_number` wraps `mpz_t`). It is
  cross-compiled to a static `libgmp.a` (see `build-gmp.sh`). Two gotchas:
  pass `CC_FOR_BUILD=/usr/bin/cc` (GMP needs a *native* compiler for build-time
  tools) and `--host=none --disable-assembly`.
- **`-DNDEBUG` is required.** Native Release defines it; Emscripten `-O2` does
  not, so without it Crab's internal `assert()`s fire at runtime.
- **No repo source edits.** Only `boxes_domain.cpp` is skipped; the apron/pk/
  pplite domain files are `#ifdef`-guarded and compile to no-ops. `boxes.hpp`
  self-guards to a dummy without `HAVE_LDD`.
- **Config shadowing.** `wasm-config/` is first on the include path so its
  `crab/config.h` (all backends off) wins over any backend-enabled config.
- **Entry point.** The full native `main()`/CLI is reused unchanged. JS writes
  the program into Emscripten's in-memory FS and invokes the CLI:
  ```js
  const M = await Crabber({ print: s => out += s + '\n', printErr: s => out += s + '\n' });
  M.FS.writeFile('/prog.crabir', source);
  M.callMain(['/prog.crabir', '-d', 'int', '--print-invariants']);
  ```
  Create a fresh `Crabber({...})` instance per run so stdout/analysis state don't
  leak between runs.
